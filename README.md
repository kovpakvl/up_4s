# SecureOffice

Настольный менеджер корпоративных аккаунтов для малого бизнеса на Python,
Tkinter и SQLite.

Приложение поддерживает два режима:

- локальное личное хранилище;
- централизованный режим компании с администратором и сотрудниками.

## Возможности

- вход по общему мастер-паролю;
- шифрование паролей через `cryptography.Fernet`;
- учёт сотрудников, ролей, отделов и статусов;
- привязка аккаунтов к сотрудникам, отделам и типам сервисов;
- дашборд со статистикой сотрудников, аккаунтов и слабых паролей;
- контроль паролей, которые не менялись более 90 дней;
- добавление, редактирование, поиск и удаление сотрудников и аккаунтов;
- защита от удаления сотрудника с закреплёнными аккаунтами;
- генератор криптографически стойких паролей;
- временный показ пароля и автоматическая очистка буфера обмена;
- автоматическая блокировка приложения;
- безопасное резервное копирование базы данных;
- автоматическая резервная копия перед сбросом хранилища;
- временная защита входа после нескольких неверных попыток.
- регистрация сотрудников по приглашению администратора;
- серверное разграничение прав;
- сотрудник видит только собственные аккаунты;
- администратор видит сотрудников и все корпоративные аккаунты.

## Структура

- `main.py` — единственная точка запуска приложения;
- `ui_login.py` — создание мастер-пароля и вход;
- `ui_main.py` — главное окно и разделы приложения;
- `ui_employee_form.py` — форма сотрудника;
- `ui_account_form.py` — форма корпоративного аккаунта;
- `auth.py` и `crypto_service.py` — проверка мастер-пароля и шифрование;
- `database.py` — создание и подключение SQLite;
- `employee_service.py` — операции с сотрудниками и отделами;
- `secureoffice_service.py` — операции с корпоративными аккаунтами;
- `settings_service.py` — настройки и резервные копии.
- `company_server.py` — центральный сервер компании;
- `company_client.py` — клиент центрального API;
- `ui_company_login.py` — подключение, регистрация и вход в компанию.

## Запуск

Рекомендуется Python 3.10 или новее.

```powershell
py -3.13 -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
.\.venv\Scripts\python.exe main.py
```

При первом запуске программа создаст базу `data/password_manager.db` и
предложит задать мастер-пароль.

## Режим компании

На компьютере администратора запустите центральный сервер:

```powershell
py -3.13 company_server.py
```

По умолчанию сервер доступен по адресу:

```text
http://127.0.0.1:8765
```

Для подключения с других компьютеров используйте локальный IP компьютера
администратора, например `http://192.168.1.10:8765`. В производственной сети
сервер должен быть опубликован только через HTTPS.

Далее:

1. В приложении нажмите `Подключиться к компании`.
2. При первом подключении создайте администратора.
3. Администратор создаёт код в разделе `Приглашения`.
4. Сотрудник выбирает регистрацию, вводит код и создаёт личный логин.
5. После входа сотрудник видит только закреплённые за ним аккаунты.

Серверная база и ключ шифрования находятся в каталоге `server_data`. Этот
каталог необходимо резервировать целиком.

## Хранение данных

Мастер-пароль не сохраняется в открытом виде. В базе хранится его PBKDF2-хеш
и случайная соль. Пароли корпоративных аккаунтов сохраняются только в
зашифрованном виде.

Резервные копии создаются в каталоге `backups`.

Файлы `app.py`, `templates` и `static` относятся к предыдущей Flask-версии и
не участвуют в запуске настольного приложения.

## Новая серверная версия

Новый вариант проекта переносится на схему:

- desktop-приложение администратора;
- backend + PostgreSQL в Docker Compose;
- браузерный кабинет сотрудника.

Запуск админского приложения:

```powershell
python admin_main.py
```

В админском приложении можно запустить сервер, создать первого администратора,
добавить сотрудника, выдать ему одноразовый ключ активации, управлять
парольными записями сотрудников и смотреть журнал действий.
