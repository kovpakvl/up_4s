# SecureOffice: пользовательская инструкция

## Для Кого

SecureOffice рассчитан на системного администратора небольшой локальной сети и сотрудников, которым нужен доступ к своим сохранённым рабочим паролям.

Администратор работает в desktop-приложении. Сотрудники работают в браузере по ссылке.

## Что Нужно Установить

На компьютере администратора:

1. Windows.
2. Docker Desktop.
3. `SecureOfficeAdmin.exe`.

На компьютерах сотрудников ничего устанавливать не нужно. Достаточно браузера и доступа к той же локальной сети.

## Первый Запуск

1. Запустите Docker Desktop.
2. Дождитесь, пока Docker перейдёт в рабочее состояние.
3. Запустите `SecureOfficeAdmin.exe`.
4. Нажмите `Запустить сервер`.
5. Дождитесь запуска контейнеров.
6. Создайте первый аккаунт администратора.
7. При желании пройдите обучение.

Первый запуск может занять несколько минут: Docker скачивает образы и собирает backend.

## Настройка Компании

Рекомендуемый порядок:

1. Откройте `Структура`.
2. Добавьте отделы.
3. Для каждого отдела добавьте должности.
4. Откройте `Сотрудники`.
5. Создайте карточки сотрудников.
6. Выдайте каждому сотруднику ключ активации.

Ключ активации создаётся индивидуально для одного сотрудника. Его можно передать вместе со ссылкой на страницу активации.

## Как Подключается Сотрудник

1. Администратор выдаёт сотруднику ссылку и ключ.
2. Сотрудник открывает ссылку `/activate`.
3. Вводит ключ, логин и пароль.
4. После активации входит через `/login`.
5. Видит только свои записи паролей.

## Работа С Паролями

Администратор может:

- создавать записи паролей для одного или нескольких сотрудников;
- редактировать записи;
- удалять записи;
- смотреть историю изменения пароля;
- видеть журнал событий.

Сотрудник может:

- видеть свои записи;
- добавлять свои записи;
- менять свои записи;
- копировать пароль;
- смотреть историю конкретной записи.

## Журнал Событий

Журнал доступен администратору. В нём отображаются понятные действия:

- вход в систему;
- создание сотрудника;
- выдача ключа;
- создание отдела или должности;
- добавление, изменение и удаление записи пароля.

Время в приложении показывается по МСК.

## Остановка Сервера

В desktop-приложении откройте `Настройки` и нажмите `Остановить`.

После остановки сотрудники временно потеряют доступ к web-кабинету. Данные остаются в Docker volume PostgreSQL.

## Частые Ошибки

### Приложение пишет, что Docker не найден

Установите Docker Desktop и перезапустите приложение.

### Docker установлен, но сервер не запускается

Запустите Docker Desktop и дождитесь статуса Running. Затем снова нажмите `Запустить сервер`.

### Сотрудник не открывает ссылку

Проверьте:

- компьютер сотрудника находится в той же сети;
- firewall не блокирует порт `8765`;
- сервер SecureOffice запущен;
- сотрудник открывает ссылку вида `http://<ip-администратора>:8765/login`.

### Ключ активации не подходит

Возможные причины:

- ключ введён с ошибкой;
- ключ уже использован;
- сотрудник уже активировал аккаунт;
- ключ перевыпущен, а сотруднику отправили старый.

## Резервное Копирование

Данные PostgreSQL хранятся в Docker volume. Для учебного проекта резервное копирование вручную можно делать средствами Docker/PostgreSQL. Перед реальным использованием стоит добавить отдельный сценарий backup/restore.
